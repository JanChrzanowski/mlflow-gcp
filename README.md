# MLflow on GCP — Cloud Run + Cloud SQL + GCS

Production-ready MLflow Tracking Server deployed on Google Cloud Platform using fully-managed Terraform infrastructure.

## Architecture

```
                  ┌──────────────────────────────────────────┐
                  │             Google Cloud Run              │
                  │                                           │
  User / Client   │  ┌────────────────────────────────────┐  │
  ───────────────►│  │     MLflow Tracking Server          │  │
  HTTPS + BasicAuth│  │   (basic-auth plugin enabled)       │  │
                  │  │                                    │  │
                  │  │  admin     → MANAGE permission     │  │
                  │  │  demo_user → EDIT   permission     │  │
                  │  └──────┬─────────────────┬───────────┘  │
                  │         │ Unix socket      │ GCS client   │
                  └─────────┼─────────────────┼──────────────┘
                            │                 │
                  ┌─────────▼──────┐  ┌───────▼────────────┐
                  │  Cloud SQL     │  │  Cloud Storage      │
                  │  (maindb)      │  │  Artifact Bucket    │
                  │  DB: mlflow    │  │  gs://…/mlflow-     │
                  │  User: mlflow  │  │  artifacts/         │
                  └────────────────┘  └────────────────────┘
```

**Security model:**
- Cloud Run exposes HTTPS only (Google-managed TLS)
- MLflow built-in `basic-auth` plugin enforces authentication on every endpoint
- `default_permission = NO_PERMISSIONS` — unauthenticated requests are rejected by MLflow
- Passwords generated with `random_password`, never stored in Terraform state — stored in **Secret Manager**
- Cloud SQL is **not publicly accessible** — connected via Cloud SQL Auth Proxy Unix socket
- Service account has minimum required IAM roles (least privilege)

> **Roadmap:** To add network-level zero-trust access, front Cloud Run with a Global Load Balancer + Identity-Aware Proxy (IAP). A `modules/load_balancer` stub is ready to be added.

## Step-by-step Deployment

### 1. Authenticate with GCP

```bash
gcloud auth login
gcloud auth application-default login
gcloud config set project YOUR_PROJECT_ID
```

### 2. Create the Terraform state bucket

```bash
export TF_STATE_BUCKET="tf-state-mlflow-$(gcloud config get-value project)"
export REGION="europe-west1"

gsutil mb -p $(gcloud config get-value project) -l ${REGION} gs://${TF_STATE_BUCKET}
gsutil versioning set on gs://${TF_STATE_BUCKET}
```

Edit `backend.tf` and set `bucket = "${TF_STATE_BUCKET}"`.

### 3. Create Artifact Registry repository

MLflow's Docker image needs a home in Artifact Registry before Terraform can reference it.

```bash
gcloud artifacts repositories create mlflow-repo \
  --repository-format=docker \
  --location=europe \
  --description="MLflow Docker images"
```

### 4. Build and push the Docker image

**PowerShell (Windows):**
```powershell
# Authenticate Docker with Artifact Registry
gcloud auth configure-docker europe-docker.pkg.dev

# Build
docker build -t mlflow:2.16.0 ./docker

# Tag
$PROJECT = gcloud config get-value project
$IMAGE = "europe-docker.pkg.dev/$PROJECT/mlflow-repo/mlflow:2.16.0"
docker tag mlflow:2.16.0 $IMAGE

# Push
docker push $IMAGE
```

**bash (Linux / macOS / WSL):**
```bash
gcloud auth configure-docker europe-docker.pkg.dev

docker build -t mlflow:2.16.0 ./docker

IMAGE="europe-docker.pkg.dev/$(gcloud config get-value project)/mlflow-repo/mlflow:2.16.0"
docker tag mlflow:2.16.0 ${IMAGE}
docker push ${IMAGE}
```

### 5. Configure terraform.tfvars

```bash
cp terraform.tfvars terraform.tfvars.local   # optional backup
```

Edit `terraform.tfvars`:

```hcl
project_id   = "your-gcp-project-id"
region       = "europe-west1"
gcs_location = "EU"
prefix       = "myco"                        # short lowercase prefix

cloudsql_instance_name = "maindb"            # existing Cloud SQL instance

mlflow_image = "europe-docker.pkg.dev/your-gcp-project-id/mlflow-repo/mlflow:2.16.0"
```

### 6. Initialise and apply Terraform

```bash
terraform init
terraform validate
terraform plan -out=tfplan
terraform apply tfplan
```

Terraform will output:

```
mlflow_url             = "https://myco-mlflow-xxxx-ew.a.run.app"
artifact_bucket        = "gs://myco-mlflow-artifacts"
cloudsql_connection_name = "my-project:europe-west1:maindb"
retrieve_admin_password_cmd = "gcloud secrets versions access latest --secret=..."
```

### 7. Retrieve credentials

```bash
# Admin password (generated by Terraform)
gcloud secrets versions access latest \
  --secret="myco-mlflow-admin-password" \
  --project=YOUR_PROJECT_ID

# Demo user password
gcloud secrets versions access latest \
  --secret="myco-mlflow-demo-password" \
  --project=YOUR_PROJECT_ID
```

### 8. Verify MLflow is running

```bash
export MLFLOW_URL=$(terraform output -raw mlflow_url)

# Health check
curl ${MLFLOW_URL}/health

# List experiments (requires credentials)
curl -u admin:ADMIN_PASSWORD ${MLFLOW_URL}/api/2.0/mlflow/experiments/list
```

## MLflow Roles & Permissions

| Role | MLflow permission | Can do |
|------|-------------------|--------|
| `admin` | `MANAGE` | Create/delete experiments, register models, manage user permissions, delete runs |
| `demo_user` | `EDIT` | Create experiments, log runs, upload artifacts, view everything — cannot delete or manage |

Permissions are set by `entrypoint.sh` at container startup via the MLflow Auth API. To add more users:

```bash
# Create a new user via MLflow API
curl -X POST \
  -u admin:ADMIN_PASSWORD \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"secret"}' \
  ${MLFLOW_URL}/api/2.0/mlflow/users/create

# Grant EDIT permission to alice
curl -X PATCH \
  -u admin:ADMIN_PASSWORD \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","permission":"EDIT"}' \
  ${MLFLOW_URL}/api/2.0/mlflow/users/update-permission
```

## Connecting from Python

```python
import mlflow

mlflow.set_tracking_uri("https://myco-mlflow-xxxx-ew.a.run.app")

# Set credentials via environment variables (do not hardcode)
# export MLFLOW_TRACKING_USERNAME=admin
# export MLFLOW_TRACKING_PASSWORD=<secret>

with mlflow.start_run(experiment_id="0"):
    mlflow.log_param("learning_rate", 0.01)
    mlflow.log_metric("accuracy", 0.95)
    mlflow.log_artifact("model.pkl")
```

Or use `.netrc` / the MLflow credentials file (`~/.mlflow/credentials`):

```ini
[mlflow]
mlflow_tracking_uri    = https://myco-mlflow-xxxx-ew.a.run.app
mlflow_tracking_username = admin
mlflow_tracking_password = <secret>
```

## Updating the MLflow image

```bash
docker build -t ${IMAGE}:NEW_TAG ./docker
docker push ${IMAGE}:NEW_TAG

# Deploy the new revision without touching infrastructure
gcloud run services update myco-mlflow \
  --image=${IMAGE}:NEW_TAG \
  --region=europe-west1
```

Terraform ignores image changes (`lifecycle.ignore_changes`) so CI/CD can deploy freely.

## Project Structure

```
mlflow-gcp/
├── main.tf                     # Root: calls all modules, enables APIs
├── variables.tf                # Root-level variable declarations
├── outputs.tf                  # Root-level outputs
├── locals.tf                   # Shared locals (labels)
├── providers.tf                # Provider + Terraform version requirements
├── backend.tf                  # GCS remote state backend
├── terraform.tfvars            # Your values (do NOT commit)
├── .gitignore
│
├── modules/
│   ├── service_account/        # SA + least-privilege IAM
│   │   ├── main.tf
│   │   ├── variables.tf
│   │   └── outputs.tf
│   ├── cloud_sql/              # References existing maindb, creates mlflow DB + user
│   │   ├── main.tf
│   │   ├── variables.tf
│   │   └── outputs.tf
│   ├── gcs/                    # Artifact bucket + lifecycle rules
│   │   ├── main.tf
│   │   ├── variables.tf
│   │   └── outputs.tf
│   └── cloud_run/              # MLflow Cloud Run service
│       ├── main.tf
│       ├── variables.tf
│       └── outputs.tf
│
└── docker/
    ├── Dockerfile              # Multi-stage, non-root, production image
    ├── requirements.txt        # Pinned Python dependencies
    ├── entrypoint.sh           # Starts MLflow + bootstraps admin/demo_user
    └── basic_auth.ini          # Auth config template (populated at runtime)
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `502 Bad Gateway` on first deploy | SSL cert provisioning takes ~15 min. Check: `gcloud compute ssl-certificates describe myco-mlflow-cert` |
| `connection refused` to Cloud SQL | Verify Cloud SQL instance name in `terraform.tfvars` and that the SA has `roles/cloudsql.client` |
| MLflow returns `401` for all requests | Check Secret Manager versions; ensure `DB_PASSWORD` env var is populated in Cloud Run |
| Cold start > 30s | Set `min_instances = 1` in `terraform.tfvars` |
| Artifacts upload fails | Confirm the GCS bucket IAM — SA needs `roles/storage.objectAdmin` on the bucket |
